This repository provides a set of tools for processing 36‑channel sensor data, extracting features, and performing both classification (Random Forest) and regression (CNN) tasks. It also includes an interactive visual tool for measuring the area occupied by a stone block in an image.

The workflow is organised as follows:

1. **Feature Extraction** – Extract statistical/spectral features from raw sensor Excel files.  
2. **Visual Area Measurement** – Interactively measure the area of a stone block using a reference calibration grid.  
3. **Sensor Data Analysis**  
   - **Random Forest Classification** – Classify impact areas using raw or extracted features.  
   - **CNN Regression** – Predict (x, y) coordinates from sensor features.  
   - **Sensor Combination Evaluation** – Assess the performance of different 4‑sensor layouts.

All dependencies are listed in `requirements.txt`.

1. Feature Extraction
Script: feature_extraction.py
This script reads raw Excel files (each with 36 columns of time‑series data) from subfolders, computes 14 statistical/spectral features per sensor (mean, std, FFT peak, etc.), and saves the features as *_features.xlsx files. The output folder structure mirrors the input.
Input folder: configurable (default excel)
Output folder: configurable (default sensor_features)
Run it before any analysis that relies on extracted features.

2. Visual Area Measurement Tool
Script: Visual_identification_of_the_area_occupied_by_the_stone_block.py
An interactive tool built with OpenCV, NumPy, Matplotlib, and PIL. It allows the user to:
Select a reference region containing 5 mm × 5 mm squares.
Automatically detect the squares and compute the pixel‑to‑millimeter scale.
Select the stone region in the image.
Calculate the stone area in pixels and square millimeters.
Blue background removal (HSV segmentation) and stone contour extraction are applied. The result is displayed on screen.

3. Sensor Data Analysis
Two independent workflows are available: one based on Random Forest classification (for area classification) and one based on CNN regression (for coordinate prediction). Each workflow includes a main training script and a script for evaluating different 4‑sensor combinations.
3.1 Random Forest – Area Classification
Main script: Random_Forest_for_Area_Classification_and_Recognition.py
Uses the raw time‑series (flattened) as input features.
Trains a Random Forest classifier with Optuna hyperparameter optimisation.
Generates t‑SNE visualisation, feature importance bar chart, and confusion matrix.
Sensor combination evaluation: Different_sensor_combinations_identification.py
Computes per‑sensor importance (averaged over the 14 features).
Creates 100 combinations of 4 sensors (top‑4 plus variants where one sensor is randomly replaced).
For each combination, trains and evaluates a Random Forest (with Optuna) on the selected sensor data.
Outputs confusion matrices (PNG), an accuracy distribution plot, and a summary CSV (top4_sensor_combination_results.csv).
Data preparation:
Raw Excel files (36 columns, no header) are organised in class subfolders (one folder per impact area). No coordinate file is required.

3.2 CNN – Coordinate Regression
Main script: Location_Recognition.py
Loads the feature files generated by feature_extraction.py.
Trains a CNN to predict (x, y) coordinates from the full 504‑feature set (36 sensors × 14 features).
Performs data augmentation, feature importance analysis, and a second training on the top‑50 most important features for comparison.
Generates training curves, scatter plots with confidence intervals, and importance heatmaps.
Sensor combination evaluation: Different_sensor_combinations_identification.py
Loads a predefined list of 4‑sensor combinations from an Excel file (s.xlsx).
For each combination, trains a CNN on the selected sensors’ features (all 56 features) and separately on a subset of the most important features (top‑50 after importance analysis).
Saves per‑combination results (R² scores, metrics) and produces a summary Excel file.
Data preparation:
Feature files from feature_extraction.py must be available in the sensor_features folder.
A coordinate file coordinate_set.xlsx is required.
